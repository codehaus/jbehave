Pending Steps

By default, JBehave does not fail with it encounters steps that do not match any method in the Steps class. These are called pending steps. This is because it is designed to allow scenario writers to write the textual scenario before the implementation (hence the behaviour-driven development). By marking a step as pending, it's telling the scenario writer that it's not yet implemented and correspondingly it will not execute any steps that following in the same scenario.

In some cases, thought, it may be useful to make the scenarios fail when steps are pending. This is done by configuring the PendingErrorStrategy in Configuration

new Configuration(){
   public PendingErrorStrategy forPendingSteps() {
       return PendingErrorStrategy.FAILING;
   }
}
