jBehave

Description

Launches the verification process and sets a property if build did not pass..

Parameters

Attribute Description Required
page The page path to the test one of the two
url The page path to the test
report The report file to generate yes
classpath the classpath to use. No
classpathref the classpath to use, given as reference to a PATH defined elsewhere. No

Nested Elements

The <fitnesse> task supports a nested <classpath> element that represents a PATH like structure.

jvmarg

Any additional parameters may be passed to the new VM via nested <jvmarg> elements. For example:

<fitnesse url="http://www.example.com/LongAcceptanceTestSuite" report="report.html">
    <jvmarg value="-Xmx256"/>
    ...

    </fitnesse>

would run the test with max heap size of 256MB, which is always a good idea when you have a long test.

<jvmarg> allows all attributes described in Command-line Arguments.

server

The Fitnesse server to connect to. This element is required if page attribute is used in fitnesse element.

This one takse the following attributes:
Attribute Description Required
root The root of the fitnesse pages yes
port The port number of the fitnesse server no (default to 80)

classpath

Java's classpath attribute is a PATH like structure and can also be set via a nested classpath element.

Examples

    <path id="jbehave.ant.lib">
        <pathelement location="${lib.build.dir}/jbehave.jar" />
        <pathelement location="${lib.build.dir}/jbehave_ant.jar" />
    </path>
    <taskdef name="jbehave" classname="jbehave.ant.JBehaveTask" classpath="jbehave.ant.lib" />
Defines the jBehave task 
    <target name="behave" depends="behaviour.compile">
        <jbehave>
            <behaviours name="jbehave.core.AllBehaviours"/>
            <classpath>
                <pathelement location="${classes.dir}"/>
                <pathelement location="${behaviour.classes.dir}"/>
            </classpath>
        </jbehave>
    </target>
