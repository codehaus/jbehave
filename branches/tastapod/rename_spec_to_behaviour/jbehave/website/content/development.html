Golden Rules

Here are some rules that we follow when developing jBehave:

  • Behaviour Driven: Never check in a class without associated Specs and Criteria.
  • Criteria Always Pass: Never check a failing criteria into the repository.
  • Acceptance Tests Can Fail: A failing acceptance test indicates something that needs to be done: a todo item, feature request or bug report, for example. It's ok to check a failing acceptance test into the CVS repository.
  • Separate Acceptance Tests: Make it easy to run only the Specs so that we can check that they pass before committing, even when acceptance tests are failing.
